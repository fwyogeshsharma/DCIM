  🚀 How to Use It

  Quick Start (5 minutes):

  Step 1: Generate Certificates
  # Linux/macOS
  ./scripts/generate-certs.sh

  # Windows
  .\scripts\generate-certs.ps1

  Step 2: Certificates will be in certs/ directory:
  certs/
  ├── ca.crt          # CA certificate
  ├── ca.key          # CA private key
  ├── server.crt      # Server certificate
  ├── server.key      # Server private key
  ├── client.crt      # Agent certificate
  └── client.key      # Agent private key

  Step 3: Configuration is already in config.yaml:
  server:
    url: "https://localhost:8443/api/v1"
    tls:
      enabled: true
      client_cert_path: "./certs/client.crt"
      client_key_path: "./certs/client.key"
      ca_cert_path: "./certs/ca.crt"
      insecure_skip_verify: false
      min_tls_version: "1.2"

  Step 4: Copy server certificates to your monitoring server
  scp certs/ca.crt server:/path/
  scp certs/server.crt server:/path/
  scp certs/server.key server:/path/

  Step 5: Configure your server for mTLS (examples in guide)

  Step 6: Run the agent
  ./network-monitor-agent -config config.yaml

  ---
  📋 Configuration Options

  Current Settings (config.yaml):

  tls:
    enabled: true                           # Enable/disable TLS
    client_cert_path: "./certs/client.crt"  # Client certificate
    client_key_path: "./certs/client.key"   # Client private key
    ca_cert_path: "./certs/ca.crt"          # CA for server verification
    insecure_skip_verify: false             # Skip cert verification (testing only)
    min_tls_version: "1.2"                  # Minimum TLS version (1.2 or 1.3)

  For Development/Testing:

  Set insecure_skip_verify: true to skip server certificate verification (not recommended for production).

  For Production:

  - Set min_tls_version: "1.3" for maximum security
  - Keep insecure_skip_verify: false
  - Use properly signed certificates
