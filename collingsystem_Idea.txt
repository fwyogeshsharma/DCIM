  3.2 ALERT DEDUPLICATION

  PROBLEM:

  If outlet temp is 65°C (above 60°C threshold), simulator sends data every 10 seconds:
  - 10:00:00 → Alert: "Outlet 65°C > 60°C"
  - 10:00:10 → Alert: "Outlet 65°C > 60°C"  ← DUPLICATE
  - 10:00:20 → Alert: "Outlet 65°C > 60°C"  ← DUPLICATE

  Result: Database flooded with duplicate alerts

  ---
  OPTION A: No Deduplication (Store All)
  // Insert every alert every time
  db.InsertAlerts(alerts)
  ✅ PROS: Simple, full history
  ❌ CONS: Database bloat, alert fatigue

  ---
  OPTION B: Suppress Duplicates (Time-Based)
  // Only insert if no identical alert in last 5 minutes
  if !db.AlertExistsRecently(alert, 5*time.Minute) {
      db.InsertAlerts(alerts)
  }
  ✅ PROS: Reduces noise
  ❌ CONS: May miss rising severity

  ---
  OPTION C: State-Based Alerts (Edges Only)
  // Insert only when alert state CHANGES
  // Rising edge: Normal → Alert
  // Falling edge: Alert → Normal

  if currentState != previousState {
      db.InsertAlerts(alerts)
  }
  ✅ PROS: Clean, event-driven
  ❌ CONS: Need state tracking

  ---
  OPTION D: Update Existing Alert
  // Find open alert of same type
  existing := db.GetOpenAlert(alert.MetricType, alert.AgentID)
  if existing != nil {
      // Update: increment counter, update timestamp
      existing.OccurrenceCount++
      existing.LastSeen = time.Now()
      db.UpdateAlert(existing)
  } else {
      // Create new alert
      db.InsertAlerts(alerts)
  }
  ✅ PROS: Tracks repeated violations
  ❌ CONS: More complex queries

      outlet_temperature DECIMAL(10,2),
